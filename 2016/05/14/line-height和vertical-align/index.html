line-height和vertical-aline

  • CSS

本文基本来自张鑫旭的博客，自己简写了许多。

line-height和vertical-aline

行内框盒子模型
  • content area(内容区域),一种围绕文字看不见的盒子，高度等于font-size大小；
  • inline boxes(内联盒子)，让内容排成一行，包含inline水平的标签（span, a, em等，例如上面的em)，则属于“内联盒子”; 如果是个光秃秃的文字，则属于“匿名内联盒子”；
  • line boxes(行框盒子)，每一行是一个“行框盒子”，每个“行框盒子”是由一个个“内联盒子”组成；
  • containing box(包含盒子)，<p>标签所在“包含盒子”，由一行行“行框盒子”(line boxes)组成。

line-height

line-height: 两行文字基线之间的距离，若未定义height，则高度由line-height决定。

内容区域高度(content area) + 行间距(vertical spacing) = 行高(line-height)

line-height: 1.5/1.5em/150%区别

  • 1.5: line-height=1.5*font-size(自己的font-size，父标签算出后，若子标签继承，则需按自己的font-siz重新计算line-height);
  • 1.5em/150%: line-height=1.5*font-size(父亲的font-size, 父标签算出后，若子标签继承，则继承父标签的line-height);

vertical-align

/*关键字值*/
vertical-align: baseline;//(默认)
vertical-align: sub;
vertical-align: super;
vertical-align: text-top;
vertical-align: text-bottom;
vertical-align: middle;
vertical-align: top;
vertical-align: bottom;

/*<长度>值*/
vertical-align: 10em;
vertical-align: 4px;

/*<百分比>值*/
vertical-align: 10%;

/*全局值*/
vertical-align: inherit;
vertical-align: initial;
vertical-align: unset;

长度值和百分比值是根据基线(baseline)来移动的，其中百分比是相对line-height的。


如上图所示，图片下面会有一段空白，why?
首先，div里面存在空白节点，相当里面有空文本。这就涉及图片和文本的对齐方式问题。由于vertical-align的默认值为baseline，当图片的底端是紧贴父元素的baseline，而文本呢？它并不是底端紧贴父元素baseline,而是比baseline还低，用过英文本子的就知道，英文本子是四线三格，其中第三条线就是baseline，第三条线到第四条线就是，上面空白的高度。看下面加了文字后就看得更清楚了。
gdh

所以要怎么解决这个问题呢？

1. 让vertical-align失效
图片默认是inline水平，而vertical-align对块状水平元素无感。因此，我们只要让图片display设为block就可以了，或者浮动或绝对定位等。例如 img{display: block;}

2. 使用其他vertical-align值
使用vertical:bottom/middle/top等，可使图片(底部/中间/上)分别于文字相应部分对齐，从而不让文本框露出(除非font-size太大)
gdh

3. 修改line-height
div{line-height:5px}
gdh

当文本行高足够小时，也可使文本框不溢出，但文字可能溢出
4. 修改font-size
line-hieght的值是由font-size计算出来的，若font-size=0,则line-height就是0，
div{font-size:0}
gdh

衍生：垂直居中

font-size=0的原因，是middle中线比不是字符内容的绝对居中，不设置会导致并不是真正的垂直居中。
t通过line-height定高，元素vertical-align:middle垂直居中的方法不仅适用于现代浏览器，连IE7也是支持的。

## inline-block和baseline
